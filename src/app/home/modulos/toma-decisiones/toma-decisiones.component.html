🗂️ Toma de Decisiones y Evaluación de Opciones

🧠 ¿Qué es la toma de decisiones?

La toma de decisiones es el proceso consciente de elegir entre varias alternativas (opciones) analizando información, premisas y consecuencias. En programación y en la vida real implica evaluar datos, aplicar reglas lógicas y seleccionar la acción más adecuada según objetivos y restricciones.

🔧 Elementos fundamentales de todo proceso de decisión

  • Opciones (alternativas): las posibilidades entre las que puedes elegir.
  • Premisas / datos: hechos o suposiciones sobre la situación (lo que sabes).
  • Criterios: lo que valoras para elegir (rapidez, coste, seguridad, calidad).
  • Restricciones: límites reales (presupuesto, tiempo, recursos).
  • Consecuencias: resultados esperados para cada opción.
  • Incertidumbre: grado de desconocimiento o probabilidad de distintos resultados.

📚 Tipos de decisiones

  • Determinísticas: resultado claro si se cumplen condiciones (ej. semáforo).
  • Probabilísticas: resultado incierto (ej. clima, mercado).
  • Simples / aisladas: una sola elección momentánea.
  • Secuenciales: múltiples decisiones encadenadas (planificación).
  • Óptimas vs. satisficing: buscar la mejor solución o una suficientemente buena (por tiempo/recursos).

🧭 Metáforas cotidianas

  • Bifurcación en el camino: llegas a un cruce y tomas la ruta según señales (condicional).
  • Carta de restaurante: eliges plato según hambre, tiempo y dinero (ponderar criterios).
  • GPS con tráfico: elige la ruta más rápida en tiempo real (evaluación dinámica con datos).
  • Comprar en oferta: comparas precio y necesidad; si el descuento compensa, compras (análisis coste-beneficio).

🔗 Cómo se conectan premisas y conclusiones

  • Premisa: “Si llueve, llevaré paraguas.”
  • Hecho observado: “Está lloviendo.”
  • Conclusión: “Llevo paraguas.”

Este tipo de razonamiento se llama Modus Ponens en lógica formal: si se acepta una premisa condicional (“Si A, entonces B”) y se verifica el antecedente (“A”), se puede concluir el consecuente (“B”).
En lógica: premisas → reglas (si… entonces, y, o, no) → conclusión. En programación esto se traduce en condiciones (if), combinaciones (&&, ||) y negaciones (!).

🧠 Modelos mentales y técnicas lógicas útiles

  • Si … entonces (implicación): estructura básica para decisiones.
  • AND (y) / OR (o) / NOT (no): combinar condiciones.
  • Árbol de decisión: representar opciones y sus resultados en ramas.
  • Función de utilidad / puntuación: asignar valor numérico a cada opción y elegir la de mayor puntaje.
  • Reglas heurísticas: atajos para decidir rápido (p. ej. “si tengo menos de 5 minutos, elijo la opción rápida”).
  • Búsqueda y optimización: cuando hay muchas opciones, aplicar estrategias (greedy, búsqueda por puntaje, backtracking).

🧾 Herramientas prácticas para evaluar opciones

1) Matriz de decisión ponderada

  1. Listar criterios relevantes.
  2. Asignar peso a cada criterio (suma 1 o 100).
  3. Puntuar cada opción por criterio (0–5).
  4. Multiplicar y sumar para obtener la puntuación total.
Opción Rapidez (0.4) Costo (0.3) Facilidad (0.3) Score total
Opción A 4 → 1.6 3 → 0.9 5 → 1.5 4.0
Opción B 3 → 1.2 5 → 1.5 2 → 0.6 3.3

2) Pros / Cons

Lista simple de ventajas y desventajas; útil para decisiones con pocos criterios.

3) Árbol de decisión básico

Dibuja nodos: en cada nodo una condición; en hojas la consecuencia o recompensa. Muy útil para decisiones secuenciales.

4) Sensitivity / Análisis de sensibilidad

Modificar pesos o supuestos para ver si la decisión se mantiene estable.

⚠️ Sesgos y errores comunes

  • Anclaje: fijarse en la primera información recibida.
  • Confirmación: buscar solo datos que confirmen nuestra idea.
  • Disponibilidad: sobrevalorar ejemplos recientes o memorables.
  • Exceso de confianza: subestimar la incertidumbre.

Enseñar a preguntar: “¿Qué supongo?” y “¿Qué pasa si esa suposición falla?” ayuda a mitigarlos.

🖥 Cómo mapear todo esto en programación

  • Condicionales (if/else): decisiones simples.
  • Switch / case: elegir entre varias alternativas discretas.
  • Funciones que devuelven puntajes: implementar matriz ponderada (score(option)).
  • Estructuras de datos (arrays / objetos): almacenar opciones, criterios y resultados.
  • Árboles (estructuras recursivas): modelar decisiones secuenciales.
  • Simulación / Monte Carlo: cuando hay incertidumbre usar simulaciones para estimar probabilidades.

Pseudocódigo ejemplo (score):

// criterios = {{ '{' }} rapidez:0.4, costo:0.3, facilidad:0.3 {{ '}' }}
            // opciones = [A,B,C]
            for opcion in opciones:
                score = 0
                for criterio in criterios:
                    score += puntaje(opcion, criterio) * criterios[criterio]
                escoger opcion con max(score)
